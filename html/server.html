How does this website work?

This website is made of a collection of HTML, CSS, and images/pdf files that are manually written in HTML   (no wix   involved, lol!). The source code is organized and stored in a git repo on my home server. A mirror is hosted by github, which I use as a remote origin: iocoder.github.io .

Taking care of the details of HTML   and CSS   helps me develop a website that can work well on most of HTML2 web browsers, and make sure it is not bloated by unnecessary piles of Javascript   and hidden content. That being said, this website doesn't have any embedded Javascript nor any client-side   code. Similarly, all the pages are purely written in the standard markup language (HTML), and no server-side   code is executed (no PHP , Ruby on Rails , etc.) to dynamically generate the web content (i.e. all the content on this website is static).

Where is this website hosted?

This website is hosted on my home server. It is stored under a directory on my disk, and web-server software is set up to listen to port 80 and port 443. The domain name ramses.ca   is registered by Canadian Internet Registration Authority   (CIRA) through an intermediate registrar.

When you type the URL ramses.ca   in your browser's address bar and hit Enter, your browser makes a DNS request   to resolve the name ramses.ca. The request is passed to my registrar   by the Internet DNS system. Next, my registrar resolves the name into my IP address. This IP address is basically the address of my landlord's gateway router at the basement of our lovely family house. All HTTP/HTTPS requests that your browser makes are then delivered to my computer upstairs through a NAT .   By visiting this page, you have just made a direct access to my home server. Congratulations!

In Layman's terms, when you open my website on your browser, your browser directly sends a request to my home computer, and my home computer answers that request by sending back the required content directly to your machine. This is basically how the Internet   works.

What are the specs of your machine?

The hardware itself is powerful enough for my use cases. I have built the machine in May 2019 (I was living in Ottawa by that time). I primarily use a Core i9 9900K   CPU. I have always been impressed by the great performance of this chip. Motherboard is Gigabyte Z390 AORUS Master   which I highly admire! It's an advanced board with artistic design.

For the RAM, I use G SKill TridentZ RGB   DDR4 units, with a total of 32GB. Most of the time, only 3GB is used by the software and the Linux Kernel. Finally, for the storage, I use a 1TB Samsung SSD 970 EVO Plus   chip. The compilation of high-performance i9 CPU, DDR4 RAM, PCIe, and M.2 SSD leaves me so satisfied with the overall performance of the machine.
